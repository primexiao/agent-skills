# explore: Multi-Perspective Topic Discussion

First apply `new-topic.md`: use `[seed]` when present, otherwise use the current conversation; merge into an existing topic or create a new one by the normal rules. Then continue the discussion below. The user may steer, skip a phase, or stop at any time.

## Explore

Select adaptive perspectives that expose genuinely different assumptions, constraints, stakeholder incentives, failure modes, or analogies. Use no fixed roles or mandatory count. Each perspective must add distinct questions, hypotheses, or counterexamples; omit perspectives that merely rename an existing point. Discuss naturally instead of dumping simulated role-play.

## Research When Needed

Research only when a factual claim affects the discussion or decision. Prefer primary sources and distinguish verified facts, inference, and unknowns. A simulated perspective is not evidence and cannot supply facts or authority.

## Decide

Periodically synthesize the strongest agreements, disagreements, options, trade-offs, and disconfirming evidence. Converge on the current position or decision when the evidence permits; otherwise state the unresolved question instead of forcing closure.

## Close

Use the existing **Session Close** rules. Update the topic `README.md` with synthesized outcomes, decisions, sources, and open questions—not the conversation transcript—then update its INDEX row and append one `log.md` line.
