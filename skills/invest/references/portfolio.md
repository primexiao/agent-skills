# portfolio — 组合审视

> 吸收自 xbtlin/ai-berkshire `portfolio-review.md` (MIT)，扩展支持黄金/BTC 持仓。

对当前持仓做组合层面审视。输入格式：`腾讯30%, 茅台20%, 黄金10%, BTC 5%, 现金35%` 或含股数/成本价的明细，或"我的持仓"（读 `~/investing/portfolio-latest.md`）。

## 第一步：解析持仓

标准化为表：标的/代码/持仓量/成本/现价/市值/占比/盈亏。只有比例时按比例分析。已有 `portfolio-latest.md` 则读取并更新。

**混合资产处理**：股票走下面的完整体检；黄金/BTC 仓位不做"论文健康度"股票式体检，改为核对 `macro.md` 的配置角色（黄金=保险，BTC=可归零风险资产）与仓位纪律，并入第四步组合分析。

## 第二步：取最新数据

- A股：`ashare_data.py quote/valuation`；港美股：WebSearch（按 `data-sources.md` 双源）
- 黄金/BTC：`macro_data.py dashboard`
- 每只股票补充：最近一季关键财务变化、近期重大事件、前瞻PE
- 估值数据用 `financial_rigor.py verify-valuation` 校验
- 每只标的标注信息丰富度（A/B/C），C级结论标低置信度

## 第三步：单仓位体检（股票）

| 标的 | 当前PE | 买入逻辑变化? | 论文健康度 | 仓位建议 |

每只回答三问：今天没持仓，当前价还会买吗？明天起不能交易，持5年舒服吗？买入论文还完整吗？

## 第四步：组合层面分析

**4.1 集中度**：第一大持仓<40%？前三大50-80%？总数5-15只？现金10-30%？（李录标准：3-5只核心占80%+，但要求每只都研究透）

**4.2 相关性**：识别隐性关联（同主题/同国别/同货币风险共振）。注意：黄金与股票通常低/负相关（这是它存在的意义）；**BTC 在风险偏好收缩时与科技股高度正相关，不要当对冲资产算**。
检查：>50%仓位暴露同一主题？同一国家/货币？中美关系恶化亏多少？全球衰退亏多少？

**4.3 机会成本**：所有股票仓位按"预期年化回报×确定性"排序。预期回报 ≈ FCF Yield + 预期增速（用 three-scenario 工具验证）。排名最后的仓位回报是否高于无风险利率？不是就该换成现金。黄金/BTC 不参与此排序（它们不是回报资产，是配置资产），但要检查配置比是否超出 `macro.md` 的纪律上限。

**4.4 压力测试**：全球衰退（盈利-20~30%）/中美冲突（中概折价50%）/利率飙升/科技泡沫破裂 + **crypto 专属情景：BTC 回撤80%（历史发生过3次）对组合的影响**。每个情景给定性+粗估量级。

## 第五步：优化建议

调仓表（加仓/减仓/清仓/新建/不动 + 理由）；现金管理建议。发现"不如现金"的仓位或想找新标的 → 建议走 `/invest check` 或 `/invest research`，不在本子命令内直接荐股。

## 第六步：输出与保存

报告结构：组合概览 → 单仓位体检 → 组合分析（集中度/相关性/机会成本/压力测试）→ 调仓建议 → 下次审视时间。

结论必须明确回答：1) 组合健康度（优秀/良好/需调整/问题严重）；2) 最应该做的一件事；3) 当前最大风险。

写入 `~/investing/portfolio-latest.md`（持仓表+结论+调仓记录追加）。**该文件含个人持仓，绝不放进任何 git 仓库或对外发送。**

## 原则

每一块钱都有机会成本；集中不是风险，无知才是；现金是一种仓位；组合层面>个股层面；每季度审视一次足够，不要过度交易。
